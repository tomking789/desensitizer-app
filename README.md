# 本地资料脱敏工具

一个本地桌面脱敏工具，用来在把资料交给 ChatGPT、Codex 或其他 AI 应用前，先在本机识别并替换敏感信息。

## 功能

- 文件脱敏：Word、Excel、文本型 PDF、普通文本文件
- 映射还原：使用本地 `mapping_*.json` 将脱敏占位符还原
- 自定义敏感词：适合补充客户名、项目名、产品名、内部系统名
- 本地运行：映射表和处理结果默认只保存在本机

## 支持格式

- Word：`.docx`
- Excel：`.xlsx`
- 文本型 PDF：`.pdf`，导出为脱敏后的 `.txt`
- 普通文本：`.txt`、`.md`、`.csv`、`.json`、`.log`、`.xml`、`.html`
- 按映射表还原：`.docx`、`.xlsx` 和普通文本文件

## 暂不支持

- 图片 OCR 脱敏
- 扫描版 PDF OCR 脱敏
- PDF 原版式覆盖式脱敏
- `.doc`、`.xls` 老格式文件

## 安装

需要 Python 3.10+。

```bash
python -m pip install -r requirements.txt
```

Windows 可以双击启动：

```text
start_desensitizer.cmd
```

也可以直接运行：

```bash
python app.py
```

## 使用流程

1. 打开“文件脱敏”页。
2. 点击“添加文件”，选择需要脱敏的文件。
3. 选择输出目录。
4. 如有客户名、项目名、内部系统名等业务敏感词，填到“自定义敏感词”，每行一个。
5. 点击“开始脱敏”。

每次脱敏会输出：

- 脱敏后的文件
- `mapping_YYYYMMDD_HHMMSS.json`
- `report_YYYYMMDD_HHMMSS.csv`

`mapping_*.json` 含原始敏感值，只能保存在本地，不要上传给 AI 或公开仓库。

## 还原流程

1. 打开“按映射还原”页。
2. 添加需要还原的脱敏文件。
3. 选择对应的 `mapping_*.json`。
4. 选择输出目录。
5. 点击“开始还原”。

## 内置识别规则

- 中国大陆手机号
- 中国大陆身份证号，带校验位验证
- 邮箱
- 银行卡号，带 Luhn 校验
- 统一社会信用代码，带校验位验证
- IPv4 地址
- URL
- 常见 API Key、token、password、private key
- 带标签的人名，例如 `姓名: 张三`、`客户姓名: 张三`
- 常见中文机构名后缀
- 合同号、订单号、项目编号、工单号等带标签编号

自动识别不可能 100% 覆盖所有敏感信息。高风险资料建议先用自定义敏感词补充客户名、项目名、产品名、内部系统名，再人工抽查脱敏结果。
