## 一、为什么需要企业定制版

通用版解决的是“文件能不能脱敏”的问题；企业定制版解决的是“员工能不能按本企业规则稳定、批量、可复核地脱敏”的问题。

很多企业在合同流转、客户资料整理、财税审计、法务协作、HR 资料处理、AI 办公和外部顾问协作中，会遇到三类常见问题：

| 常见问题 | 通用版的局限 | 企业定制版的价值 |
|---|---|---|
| 员工不知道哪些词必须脱敏 | 需要每次手动输入，容易漏掉公司简称、项目代号、客户名、高管姓名等 | 企业专属词库内置到安装包，员工打开即可使用 |
| 不同部门规则不一致 | 每个人临时判断，输出文件风格不统一 | 将企业规则、行业字段、替换格式统一配置 |
| 外发后需要内部复核 | 手动记录映射关系成本高，也容易丢失 | 输出加密映射表，持有映射表和密码的人可还原核对 |
| AI 分析、审计抽样、外部协作风险高 | 文件直接外发或上传，敏感信息容易暴露 | 本地先脱敏，再给 AI 或外部人员使用 |

企业定制版不是简单换 Logo，而是把企业自己的标识、敏感词库、行业字段、脱敏策略和交付文档固化到工具里。

## 二、企业定制版可以定制什么

### 1. 专属品牌与软件入口

- 软件标题显示企业专用版名称，例如“XX 集团资料脱敏专用版”。
- 主界面展示企业简称、版本名称和企业专属 Logo。
- 关于页显示客户企业信息、版本类型和内置词库数量。
- 可按企业使用场景命名版本，例如“内网部署版”“审计专用版”“AI 办公脱敏版”。

### 2. 企业定制化敏感词库

企业版可以预置企业自己的敏感词，减少员工每次手动补词的成本。

| 词库类型 | 示例 |
|---|---|
| 企业身份 | 企业全称、简称、英文名、统一社会信用代码、邮箱域名 |
| 品牌与产品 | 品牌名、产品名、系统名、平台名、内部工具名 |
| 人员与组织 | 高管姓名、部门名称、项目负责人、关键岗位 |
| 业务对象 | 客户名称、供应商名称、合作伙伴名称、渠道名称 |
| 项目信息 | 项目代号、合同编号规则、内部项目编号、投标项目名 |
| 行业字段 | 凭证号、发票号、客户号、员工号、病历号、案号、物料号等 |

### 3. 行业规则包

可根据客户行业扩展默认识别字段，减少“通用规则识别不到业务字段”的问题。

| 行业/部门 | 可扩展识别内容 |
|---|---|
| 财税审计 | 凭证号、发票号、纳税人识别号、银行账号、客户/供应商名称、抽样编号 |
| 法务/合规 | 案号、合同主体、律师/律所、法院、仲裁机构、合同编号 |
| HR/行政 | 员工号、身份证号、手机号、薪酬字段、部门、岗位、候选人信息 |
| 金融 | 客户号、账号、交易流水号、银行卡号、风控标签 |
| 医疗 | 患者编号、病历号、医保号、检查单号、科室与医生信息 |
| 制造业 | 供应商编码、物料号、BOM 编号、图纸编号、工单号、项目编号 |

### 4. 脱敏策略

- 姓名可替换为“人员001/人员002”。
- 公司名可替换为“机构001/客户001/供应商001”。
- 手机号可完全替换，也可按企业要求保留前三后四。
- 身份证号、银行卡号、账号、邮箱可按规则替换。
- 项目代号、产品名、客户名可按企业词库统一替换。
- Excel 中同名不同主体可按员工编号、客户编号、手机号、邮箱、身份证号等主体键区分。
- 可选择可还原脱敏或不可还原脱敏，适配外发、审计复核、归档等不同流程。

### 5. 部署与交付

- 企业专用安装包。
- 企业配置包，包括 `profile.json`、`terms.csv`、`terms.txt`、`logo.png`。
- 企业词库模板和管理员维护说明。
- 使用说明、内部推广说明和示例文件。
- 可选内网/离线部署。
- 可选年度词库维护、规则调优和版本更新服务。

## 三、内部审计场景怎么用

内部审计场景的核心不是“把所有信息删掉”，而是在不影响审计分析的前提下，把不该外泄的主体信息替换成稳定占位符。

### 典型使用场景

| 审计工作 | 脱敏前风险 | 企业版处理方式 |
|---|---|---|
| 抽样凭证交给外部顾问复核 | 凭证中包含客户名、供应商名、银行账号、经办人、手机号 | 先按企业词库和财税审计规则脱敏，再外发 |
| 客户/供应商清单做异常分析 | 名称、联系人、电话、邮箱、账号直接暴露 | 将主体替换为客户001、供应商001、人员001 |
| 合同和访谈纪要交给 AI 总结 | 合同主体、项目代号、联系人和内部系统名泄露 | 本地脱敏后再给 AI 做摘要、分类、风险点提取 |
| 审计底稿跨部门流转 | 不同人员能看到超出权限的原始信息 | 输出脱敏版用于流转，原文映射表由企业指定人员保存 |
| 后续问题回查 | 脱敏后无法知道“客户001”是谁 | 使用脱敏时生成的映射表和密码还原 |

### 推荐流程

1. 审计或财务管理员维护企业词库，例如客户、供应商、项目代号、高管、银行账户关键字段。
2. 员工打开企业专用版，企业词库自动出现在“业务敏感词”区域。
3. 员工添加凭证、合同、客户清单、访谈纪要、审计底稿等文件。
4. 点击“扫描候选信息”，系统识别个人信息、企业词库和行业字段。
5. 员工确认候选项，必要时调整替换值。
6. 点击“开始脱敏”，输出脱敏文件、处理报告和加密映射表。
7. 脱敏文件用于外部顾问协作、AI 分析、跨部门流转或归档预览。
8. 如需核对原文，使用脱敏时生成的映射表和密码在“按映射还原”页还原。

### 审计场景的实际好处

- 外部顾问或 AI 能看到业务结构，但看不到真实客户、供应商、人员、账号等敏感信息。
- 同一个客户或供应商在多份文件里保持一致占位符，便于统计、筛选和关联分析。
- 两个同名人员或同名客户可以通过编号、手机号、邮箱等主体键区分，减少误合并。
- 原始资料默认留在企业环境内，减少资料出域风险。
- 加密映射表由企业自行保存，满足“可复核但不随意暴露”的内控要求。

## 五、给企业带来的价值

### 1. 降低资料外发和 AI 使用风险

合同、客户资料、会议纪要、审计底稿、员工资料在交给 AI 工具、外部顾问、合作方或跨部门流转前，可以先在本地完成脱敏。

### 2. 减少员工操作负担

企业词库默认内置，员工不需要每次记住公司全称、简称、产品名、项目代号、高管姓名、客户名等敏感词。

### 3. 让输出结果更统一

企业可以统一占位符风格，例如“人员001”“客户001”“供应商001”“项目001”，便于后续阅读、统计和归档。

### 4. 支持可还原和可复核

对审计、法务、财务等需要后续核对的场景，可以保留加密映射表。持有映射表和密码的人可以还原，普通外发文件仍保持脱敏状态。

### 5. 形成内部数据安全流程

企业可以把“先脱敏、再外发/再给 AI/再归档”写入内部制度，减少员工临时判断带来的不一致。

### 6. 适合内网和离线环境

对数据出域要求较高的客户，可采用内网版或离线部署，原始资料、映射表和还原密码由企业自行管理。

## 六、员工使用流程

1. 打开企业专用版软件。
2. 添加需要脱敏的 Word、Excel、PowerPoint、PDF 或文本文件。
3. 查看“业务敏感词”区域，企业内置词库会在启动时自动写入。
4. 保留默认企业词库，必要时追加本次任务的临时词汇。
5. 点击“扫描候选信息”。
6. 在候选表中确认需要脱敏的内容，可启用、禁用、删除或修改替换值。
7. 点击“开始脱敏”。
8. 将输出的脱敏文件用于 AI 分析、外部协作、跨部门流转或审计归档。
9. 保存处理报告和加密映射表；如后续需要还原，使用脱敏时设置的映射表密码。

## 七、管理员维护流程

1. 使用 `enterprise/terms.csv` 维护企业专属词库。
2. 表头建议使用：`词汇(value), 类型(entity), 分类(category), 备注(note), 启用(enabled)`。
3. 使用 `enterprise/profile.json` 配置企业名称、软件标题、版本名称、横幅文案和 Logo 路径。
4. Logo 默认放在 `enterprise/logo.png`，也可在 `profile.json` 中指定其他路径。
5. 正式配置放入安装目录的 `enterprise` 文件夹后，软件启动会自动读取。
6. 词库变更后，可通过新安装包交付，也可由管理员在软件界面中导入 CSV/TXT 补充。
7. 对高风险资料，建议要求员工先扫描、再人工确认候选项。

## 八、企业需要提供的资料

为完成企业定制版交付，建议客户提供：

- 企业全称、简称、英文名。
- 企业 Logo 文件。
- 产品名、品牌名、系统名、内部平台名。
- 需要默认脱敏的高管姓名、部门名称、项目代号。
- 常见客户、供应商、合作伙伴名称。
- 行业字段示例，例如客户号、员工号、合同号、发票号、凭证号、项目编号。
- 期望的脱敏显示方式，例如“人员001”“客户001”“某公司”。
- 是否需要可还原、内网部署、离线部署或管理员维护说明。

## 九、Logo 提供要求

- PNG 格式优先，也可提供 GIF。
- 透明背景优先。
- 横向 Logo 优先，适合放在软件顶部企业横幅。
- 建议宽度不低于 512px。
- 文件大小建议小于 2MB。
- 如有品牌色规范，可一并提供主色、辅助色和禁用规则。
